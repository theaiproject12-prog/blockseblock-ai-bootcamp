ℹ What is CAG? CAG (Cache-Augmented Generation) skips chunking entirely — all documents are loaded into one big context window and the model reads everything. No vector DB, no retrieval step, zero missed chunks. Works best for <50 pages of stable documents. RAG (what we build here) scales to many documents but requires retrieval to work correctly.
